Problem Definition

Task: Predict house sale prices from structural and design features

Metric: Root Mean Squared Log Error (RMSLE) = √(MSE(log(y), log(ŷ)))

Rationale: RMSLE penalizes percentage errors equally across price ranges, handling heteroscedasticity

Feature Engineering (14 New Features)

  1. HouseAge = YrSold - YearBuilt → Age at time of sale (not calendar year)
  2. RemodAge = YrSold - YearRemodAdd → Years since last remodel
  3. GarageAge = YrSold - GarageYrBlt → Garage age (Age = 0 if integrated, Age < 0 if added later)
  4. YearsSinceRemod = max(0, YearRemodAdd - YearBuilt) → Remod frequency indicator
  5. TotalSF = TotalBsmtSF + 1stFlrSF + 2ndFlrSF → Total living space
  6. TotalPorchSF = WoodDeckSF + OpenPorchSF + EnclosedPorch + ScreenPorch → Outdoor space
  7. TotalBath = FullBath + 0.5×HalfBath + BsmtFullBath + 0.5×BsmtHalfBath → Total bathroom count
  8. HasGarage = (GarageArea > 0) → Binary garage indicator
  9. HasFireplace = (Fireplaces > 0) → Binary fireplace indicator
  10. Has2ndFloor = (2ndFlrSF > 0) → Binary second floor indicator
  11. HasBsmt = (TotalBsmtSF > 0) → Binary basement indicator
  12. HasWoodDeck = (WoodDeckSF > 0) → Binary wood deck indicator
  13. LivingLotRatio = GrLivArea / LotArea → Space utilization ratio
  14. BsmtRatio = TotalBsmtSF / TotalSF → Basement proportion

Transformations

  • Log1p Transformation: Applied to 22 highly skewed features (|skew| > 0.75) for normalization
  • Target Encoding: High-cardinality features (Neighborhood, Exterior1st, Exterior2nd, SaleType) encoded via smoothed means (α=10)
  • One-Hot Encoding: Remaining categorical variables converted to binary features
  • StandardScaler: Numeric features scaled to zero mean, unit variance for linear models

  • Structural Features Dominate: OverallQual, GrLivArea, TotalSF consistently top-ranked (30-40% relative importance)
  • Age Matters: HouseAge, YearBuilt, YearRemodAdd ranked 6-8, confirming temporal effect on valuation
  • Engineered Features Valuable: TotalSF (synthetic) ranks 3rd, TotalBath (5th) — validates feature engineering
  • Location Encoded: Neighborhood (target-encoded) ranks 10th, indicating geographic price variation
  • Garage & Lot Size: GarageArea (4th), LotArea (9th) confirm external amenities importance

💡 Key Findings & Insights

Data Quality & Preprocessing

  • 2 outliers removed: GrLivArea > 4000 sf with SalePrice < $300k (anomalies)
  • Missing values strategic: Domain knowledge → 'None' for absent features, 0 for absent areas
  • Target skewness: Log transformation reduces skew from ~1.88 to ~0.15 (nearly normal)
  • Feature engineering impact: 14 new features improve ensemble models by 3-5% RMSLE

Model Selection Rationale

  • Ensemble > Linear: Gradient boosting models 13-14% better than Ridge/Lasso (0.1055 vs 0.1243)
  • Captures Non-Linearity: Random Forest (0.1134) demonstrates threshold effects (e.g., OverallQual ≥ 8 jump)
  • Gradient Boosting Best-in-Class: Learns residual errors sequentially; LightGBM/XGBoost leverage leaf-wise growth
  • Regularization Important: L2 (Ridge) handles GarageCars/GarageArea multicollinearity; L1 (Lasso) selects 132/148 features

Production Deployment

  • Model Choice: LightGBM recommended (RMSLE: ~0.105, fastest training)
  • Ensemble Strategy: Average predictions from LightGBM + XGBoost for robustness
  • Prediction Post-processing: Apply expm1(log_prediction) to recover original SalePrice scale
  • Hyperparameter Tuning: Bayesian optimization can further reduce RMSLE to ~0.10
